# Claim Boundaries for OpenAI Manufacturing Content

## Safe claims

- The repository implements five bounded manufacturing routes.
- The public runtime is Docker-runnable and exposes a FastAPI Web/API surface.
- The project supports in-memory and Qdrant vector-store profiles.
- Optional OPEA-compatible and TEI embedding profiles are present.
- The current LLM boundary supports an OpenAI-compatible Chat Completions-shaped endpoint.
- The default reproducible explanation path is deterministic.
- The committed lightweight route evaluation reports 15/15 cases passing.
- The proposed native OpenAI layer is an implementation roadmap.

## Claims requiring additional evidence

- Production use of an actual OpenAI model endpoint.
- OpenAI model latency, throughput, cost, or quality in the five routes.
- Production multimodal performance on cleared factory image sets.
- Customer deployment scale or business ROI.
- Safety certification or autonomous release authority.
- Official OPEA acceptance of unmerged contributions.

## Prohibited wording without new evidence

- “OpenAI already powers the production factory workflow.”
- “The 3.99 ms benchmark is OpenAI inference latency.”
- “The AI can approve restart, quality release, or safety clearance.”
- “The system is a certified industrial safety controller.”
- “All 20 external platforms have been published” without verified public URLs.
