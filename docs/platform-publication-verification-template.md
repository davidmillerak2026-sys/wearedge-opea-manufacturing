# Platform Publication Verification Template

Use one block per public platform after publication.

```yaml
platform: ""
account: ""
language: ""
title: ""
public_url: ""
published_at_local: ""
verified_at_utc: ""
canonical_url: ""
source_asset: ""
status: "Published"
notes: ""
```

Verification requirements:

1. The URL opens without an owner-only draft session.
2. The title and author/account are correct.
3. The article body is present, not merely an empty placeholder.
4. Repository and/or canonical links resolve.
5. No private plant data, credentials, or unapproved images are exposed.
6. Current implementation and proposed OpenAI-native roadmap remain distinct.
7. Any benchmark number retains its explicit claim boundary.

Do not record an editor URL, preview URL, screenshot, draft ID, or attempted submission as `Published`.
